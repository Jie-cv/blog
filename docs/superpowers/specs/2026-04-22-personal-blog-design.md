# 个人博客前端页面设计文档

- 日期：2026-04-22
- 技术方向：React + GSAP
- 页面类型：单页个人博客首页
- 风格关键词：科技感、成长感、叙事感、目标感

## 1. 目标

构建一个用于展示个人经历、成长路径、项目能力和博客内容的前端页面。页面不是传统简历展示，而是以“个人品牌首页 + 成长历程时间轴 + 思考/博客入口”的方式，完整表达用户从高考失利、专业受阻，到主动转向前端、获得第一段实习、坚定冲击大厂的成长主线。

页面需要同时满足以下目标：

1. 第一眼建立清晰的个人形象和技术方向。
2. 用叙事方式展示关键成长节点，而不是平铺履历。
3. 通过项目和技术栈区块体现能力证明。
4. 兼容后续扩展为持续更新的个人博客首页。
5. 借助 GSAP 动画增强页面节奏和科技感，但不影响阅读。

## 2. 用户画像与访客预期

目标访客主要包括招聘方、面试官、技术同行以及未来回看页面的自己。

访客在浏览页面时需要快速得到以下信息：

- 你是谁，现在处于什么阶段。
- 你为什么选择前端，以及这个决定是如何形成的。
- 你经历过哪些关键转折。
- 你是否有实际项目、技术积累和持续输出能力。
- 你未来想往哪里走。

因此页面设计应以“高信息密度 + 清晰阅读节奏 + 强记忆点”为核心，而不是单纯追求特效。

## 3. 页面定位

该页面定位为“科技感叙事型单页主页”。

与普通博客首页相比，它更强调个人成长故事；与普通在线简历相比，它更强调节奏、叙事和技术表达。页面最终要传达的不是“我会什么”的堆砌，而是“我是如何一步一步走到这里，并且会继续往前走”的连续感。

## 4. 信息架构

页面采用单页结构，自上而下分为 6 个区块：

### 4.1 Hero 首屏

作用：第一眼建立个人品牌印象。

包含内容：

- 姓名
- 当前身份标签（大三 / 前端开发方向 / 持续成长）
- 一句核心定位文案
- 简短补充说明
- 两个主按钮：查看成长历程、查看博客/思考
- 右侧科技感视觉模块

建议文案方向：

- 从非科班转轨到前端开发，我在持续构建自己的技术路径。
- 我不是一路顺利，但我一直在往前走。

Hero 区承担“吸引”和“定调”职责，不放过多细节。

### 4.2 About 当前状态

作用：用最短篇幅说明当前阶段。

包含内容：

- 当前为大三学生
- 学校和专业背景
- 目前主攻前端开发
- 当前目标：继续深耕 React 体系，积累实习与工程经验，冲击更高平台

这一部分作为首屏和时间轴之间的过渡，用于帮助访客快速建立背景认知。

### 4.3 Journey 成长历程时间轴

作用：作为整页核心内容，展示成长路径和关键节点。

建议节点顺序如下：

1. 高中阶段：理科生，就读南丰县第一中学。高考前 3 个月从约 200 名提升到全校 68 名，建立强烈的自驱意识。
2. 高考失利：理想中的 211 未能实现，第一次明确产生“要靠后续努力翻盘”的想法。
3. 志愿与专业偏差：填报时原本倾向网络空间安全和计算机，却意外写入工商管理；入学后原计划通过转专业靠近技术方向。
4. 转专业受阻：大一下册准备转专业时政策变化，原专业不再允许转入目标方向，第一次正面面对“计划失效”。
5. 转入 GIS：虽然没有直接转到计算机，但选择转入仍涉及开发学习的 GIS 专业，为后续技术道路寻找落点。
6. 路线迷茫：开始反复思考考研还是就业、非科班是否有机会、双非是否能进入大厂、真实就业环境如何。
7. 2025-02-11 开始学习前端：因为前端“看得见、反馈强、有意思”，开始持续投入，并且在长期学习中依然保持兴趣。
8. 被比赛短暂打断：中途因老师要求参与 C# 相关比赛发生动摇，但最终重新回到前端学习，确认真正想走的方向。
9. 高压并行阶段：补修转专业前课程、完成当前专业课程、自学技术同时进行，原本计划在 6 月开始找实习，又被学校安排测量任务拖延到 7 月 11 日。
10. 第一段实习：通过不断投递，进入华顺信安，开始接触真实团队协作、公司运转和职业视角。
11. 路线进一步清晰：与 leader 交流后，坚定“先把前端学深，再把 Node 补上”的方向，不再摇摆于是否先转向后端。
12. 目标落点：得知工位前任进入字节后，暗自定下进入字节的目标，从“进大厂”变为“进字节”。
13. 认知变化：有了第一段实习后，开始意识到双非与非科班并非决定性限制，就业环境也不是完全没有机会，因此决定不再考研，而是专注技术与实习。
14. 继续冲刺：回校后学习 React 及相关生态，只投大厂，前期不断受挫，后期开始获得机会，并最终接到字节面试电话。

时间轴区块要重点突出以下 4 个高光节点：

- 2025-02-11 开始学习前端
- 第一段实习：华顺信安
- 定下进入字节的目标
- 接到字节面试电话

### 4.4 Skills / Projects 能力与项目区

作用：承接故事之后的能力证明。

包含内容：

- 技术栈标签：React、JavaScript、TypeScript、前端工程化、组件化、Node 等
- 项目卡片 2~4 个
- 每个项目包含：项目名、简介、技术栈、个人职责、亮点

该区块当前允许先以结构占位，具体项目文案后续根据简历或补充信息完善。

### 4.5 Thinking + Blog 思考与博客区

作用：兼顾“阶段思考卡片”和“真实博客入口”。

建议拆为两列：

左侧为“阶段思考”：

- 为什么我最后坚定走前端
- 第一段实习之后，我对成长路线的重新理解
- 非科班 / 双非背景下，我如何建立信心

右侧为“博客文章入口”：

- React 学习笔记
- 前端实习总结
- Fiber 架构学习记录
- 面试复盘 / 技术总结

这样即使当前没有完整文章列表，页面也依然成立；后续新增文章时可以自然扩展。

### 4.6 Footer 结尾宣言

作用：收束页面情绪并留下记忆点。

建议呈现：

- 一句短宣言
- 简单联系方式或社交入口

建议文案方向：

- 我走得不算顺，但我一直在向前。
- 从一次次偏离，到越来越坚定地走向前端。
- 目标不会自己靠近，但我会继续往前走。

## 5. 视觉设计方向

整体视觉采用“深色科技感”路线，但强调克制和可长期使用。

### 5.1 配色

建议配色特征如下：

- 主背景：深蓝黑 / 近黑色
- 强调色：电蓝、青蓝、紫蓝渐变
- 正文：低饱和高对比白色
- 辅助文字：灰蓝色
- 点缀：少量高亮青色

目标是形成“沉稳、未来、技术化”的视觉感受，而不是廉价炫光。

### 5.2 组件风格

组件风格统一采用：

- 中等圆角
- 轻描边
- 微弱发光阴影
- 深浅层次分明的卡片背景
- 大标题配合高层级间距

应避免：

- 多色混用
- 重阴影
- 过多发光边框
- 过度装饰图标

### 5.3 布局特征

- Hero 区左文右图
- 内容区使用大留白和清晰栅格
- 时间轴区采用纵向节点式布局
- 卡片区采用响应式网格

## 6. GSAP 动画设计

页面使用 GSAP + ScrollTrigger 作为主要动画方案。动画的目标是增强叙事节奏，而不是制造噪音。

### 6.1 首屏入场动画

页面加载时：

- 主标题从下向上淡入
- 副标题延迟出现
- 标签和按钮 stagger 依次出现
- 右侧视觉模块轻微缩放并由模糊过渡为清晰

### 6.2 背景氛围动画

可加入轻量动态背景：

- 光晕缓慢漂移
- 网格或线条轻微移动
- 少量粒子浮动

该动画必须足够轻，不能影响主阅读区域。

### 6.3 时间轴滚动动画

作为页面核心交互：

- 节点进入视口时从左右滑入并淡入
- 节点圆点和连接线被逐步点亮
- 当前滚动节点高亮显示
- 关键节点使用更明显的脉冲或扫光强调

### 6.4 卡片交互动效

用于项目卡片、思考卡片、博客卡片：

- hover 时轻微上浮
- 边框或阴影增强
- 局部渐变偏移
- 箭头或图标轻微位移

### 6.5 区块级入场节奏

每个 section 在进入视口时：

- section 标题先出现
- 内容元素再 stagger 出现
- 避免整个区块一次性硬切显示

### 6.6 动画边界

明确避免以下问题：

- 不做全屏乱飞粒子
- 不做强 3D 翻转
- 不做严重影响阅读的视差
- 不让所有元素同时运动
- 不让动画成为页面主角

## 7. 文案语气与叙事策略

文案整体采用“真诚、克制、有力量”的语气。

原则如下：

1. 不卖惨，但允许表达困难。
2. 不自夸，用事实展示成长。
3. 强调“选择、坚持、修正路径”的过程。
4. 每个关键节点都尽量体现一个认知变化。

页面中的人物叙事要突出以下价值：

- 自驱力强
- 遇挫后会重建路径
- 有长期投入能力
- 愿意从现实反馈中修正判断
- 对技术方向越来越坚定

## 9. 响应式要求

页面至少适配以下场景：

- 桌面端：主要展示场景
- 平板端：时间轴和卡片区合理折行
- 手机端：单列布局，时间轴改为紧凑竖向阅读

移动端需要重点保证：

- Hero 不拥挤
- 时间轴节点可读
- 动画在性能和节奏上不过度

## 10. 非目标

本阶段不包含以下内容：

- 后台管理系统
- 真实博客 CMS 接入
- 评论系统
- 登录系统
- 复杂 3D 场景
- 过度营销式视觉包装

初版目标是做出一个完成度高、叙事清晰、可继续扩展的个人前端主页。

## 11. 后续实现建议

后续实现阶段建议先完成如下顺序：

1. 搭建 React 页面结构
2. 完成基础样式系统与页面布局
3. 落地时间轴与卡片区内容
4. 接入 GSAP 与 ScrollTrigger
5. 调整响应式体验
6. 最后补项目文案、博客入口和细节润色

## 12. 当前已知约束

- 当前工作目录为空目录，需要从零开始搭建前端项目。
- 用户已明确技术要求为 React。
- 用户已明确需要加入 GSAP 动画。
- 用户已明确页面需要同时包含个人品牌首页、成长历程、思考卡片和博客入口。
- 当前环境无法直接读取 PDF 简历内容，因此项目区具体文案需要后续根据用户补充内容或可读格式简历继续完善。
